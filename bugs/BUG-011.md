# BUG-011: Sessions Not Broadcasting Media Correctly

| Field | Value |
|-------|-------|
| **Bug ID** | BUG-011 |
| **Severity** | High |
| **Priority** | P1 |
| **Feature Area** | Media Playback / WebSocket |
| **Status** | ✅ Fixed & Verified |

---

## Description

When host plays media, only some participants receive playback events. Others see frozen or buffering state.

---

## Root Cause

Race condition in connection management - broadcast sending to random subset of connections.

---

## Fix Applied

Added mutex lock:

```go
func (sm *SessionManager) BroadcastToSession(sessionID string, message interface{}) {
    sm.mu.Lock()
    defer sm.mu.Unlock()
    
    session := sm.sessions[sessionID]
    for _, user := range session.Users {
        if user.Conn != nil {
            user.Conn.WriteJSON(message)
        }
    }
}
```

---

## Verification

✅ 8 concurrent participants  
✅ All receive play events  
✅ No missing broadcasts

---

**Status:** ✅ Fixed
