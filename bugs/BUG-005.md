## Description

When users refresh during an active 3D cinema session, the media player resets to 0:00 while others continue watching at the correct timestamp.

---

## Root Cause

Media state stored only in React state - page refresh clears it. No server-side persistence.

---

## Fix Applied

WebSocket broadcasts every 2 seconds + database persistence:

```go
func (sm *SessionManager) BroadcastMediaState(sessionID string) {
    message := map[string]interface{}{
        "type": "media_state_update",
        "current_time": session.CurrentMediaTime,
        "is_playing": session.IsPlaying,
    }
    sm.BroadcastToSession(sessionID, message)
    db.Model(&models.Session{}).Where("session_id = ?", sessionID).
       Update("current_media_time", session.CurrentMediaTime)
}
```

---

## Verification

✅ Video at 10:00, participant refreshes  
✅ Resumes at 10:00 immediately  
✅ All users synced within 1 second
