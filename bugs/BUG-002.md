## Description

Users could join paid watch sessions without purchasing tickets by exploiting missing ticket validation in session join logic. Frontend showed "Purchase Ticket" button, but backend WebSocket connection did not verify ticket ownership before allowing access.

**Business Impact:**
- Direct revenue loss (estimated ₦500 per bypassed session)
- Unfair to paying users
- Business model broken (no incentive to pay)

---

## Root Cause

**Location:** `backend/handlers/websocket_handler.go` - Line 87  
**Issue:** WebSocket upgrade did not validate ticket ownership

---

## Fix Applied

```go
func HandleSessionWebSocket(c *gin.Context) {
    sessionID := c.Param("session_id")
    userID := c.GetInt("user_id")
    
    var session models.Session
    db.First(&session, "session_id = ?", sessionID)
    
    // Validate ticket ownership for paid sessions
    if session.TicketPrice > 0 && session.HostID != userID {
        var ticket models.Ticket
        err := db.Where("user_id = ? AND session_id = ? AND status = ?", 
                       userID, sessionID, "valid").First(&ticket).Error
        if err != nil {
            c.JSON(403, gin.H{"error": "Ticket required"})
            return
        }
    }
    
    conn, _ := upgrader.Upgrade(c.Writer, c.Request, nil)
    sessionManager.AddUser(sessionID, userID, conn)
}
```
