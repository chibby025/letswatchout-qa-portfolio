# BUG-009: User Join Issue - Members Modal Not Capturing Participants

| Field | Value |
|-------|-------|
| **Bug ID** | BUG-009 |
| **Severity** | High |
| **Priority** | P1 |
| **Feature Area** | Session Management / UI |
| **Status** | ✅ Fixed & Verified |

---

## Description

Session members modal displays incorrect participant count and missing usernames. Shows "0 members" despite active participants.

---

## Root Cause

WebSocket join events not updating frontend participant state array.

---

## Fix Applied

Added WebSocket listener:

```jsx
socket.on('participant_update', (data) => {
  setParticipants(data.participants);
});
```

---

## Verification

✅ 5 concurrent users joined  
✅ Modal shows correct count and names  
✅ Real-time updates on join/leave

---

**Status:** ✅ Fixed
