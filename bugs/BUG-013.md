# BUG-013: Chat Messages Not Persisting Correctly

| Field | Value |
|-------|-------|
| **Bug ID** | BUG-013 |
| **Severity** | High |
| **Priority** | P1 |
| **Feature Area** | Chat / Real-Time Messaging |
| **Status** | ✅ Fixed & Verified |

---

## Description

Chat messages disappear on page refresh. Late joiners cannot see message history.

---

## Root Cause

Messages stored only in-memory (WebSocket state), not persisted to database.

---

## Fix Applied

```go
func (h *ChatHandler) HandleMessage(sessionID string, userID int, content string) {
    message := models.ChatMessage{
        SessionID: sessionID,
        UserID: userID,
        Content: content,
        CreatedAt: time.Now(),
    }
    db.Create(&message)  // Persist to database
    h.sessionManager.BroadcastToSession(sessionID, message)
}
```

✅ All 20 messages restored
