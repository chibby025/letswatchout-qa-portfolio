# BUG-010: Session Exit/Rejoin Logic Broken

| Field | Value |
|-------|-------|
| **Bug ID** | BUG-010 |
| **Severity** | High |
| **Priority** | P1 |
| **Feature Area** | Session Management |
| **Status** | ✅ Fixed & Verified |

---

## Description

Users who leave and rejoin sessions encounter "Session already joined" errors or WebSocket connection failures.

---

## Root Cause

Session cleanup on leave incomplete - user remains in active users array.

---

## Fix Applied

```go
func (sm *SessionManager) HandleUserLeave(sessionID string, userID int) {
    sm.RemoveUser(sessionID, userID)
    if conn := sm.getUserConnection(userID); conn != nil {
        conn.Close()
    }
    sm.BroadcastToSession(sessionID, map[string]interface{}{
        "type": "user_left",
        "user_id": userID,
    })
}
```

---

## Verification

✅ User leaves session  
✅ User rejoins immediately  
✅ No errors, connection successful

---

**Status:** ✅ Fixed
