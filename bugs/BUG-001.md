## Description

The age restriction system allowed minors (users under 18) to view 18+ rated content by bypassing API validation. Frontend filtering existed but backend did not validate user age against content rating when serving content.

**Business Impact:**
- Legal compliance risk (child safety regulations)
- Platform reputation damage
- Potential regulatory fines
- User trust erosion

---

## Root Cause

**Location:** `backend/handlers/content_handler.go` - Line 142  
**Issue:** Content query did not include age validation JOIN

```go
// Original code (vulnerable)
func GetAvailableContent(c *gin.Context) {
    var sessions []models.Session
    db.Where("is_active = ?", true).Find(&sessions)  // No age check
    c.JSON(200, sessions)
}
```

---

## Fix Applied

Added backend middleware that calculates user age from DOB and enforces filtering at the database query level:

```go
func AgeRestrictionMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        userID := c.GetInt("user_id")
        var user models.User
        if err := db.First(&user, userID).Error; err != nil {
            c.AbortWithStatusJSON(401, gin.H{"error": "Unauthorized"})
            return
        }
        age := calculateAge(user.DateOfBirth)
        c.Set("user_age", age)
        c.Next()
    }
}
```
